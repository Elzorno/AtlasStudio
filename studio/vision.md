# AtlasStudio Vision

## Mission

AtlasStudio coordinates human creators and specialized AI agents to build complete, playable games.

It acts as the creative director, production coordinator, canon authority, and quality gate for a project.

## Why AtlasStudio Exists

Atlas v1 successfully created structure, but it became too focused on generating artifacts about the game instead of producing the game itself.

AtlasStudio changes the center of gravity:

- From screens to worlds.
- From documents to playable builds.
- From single-agent execution to multi-agent production.
- From RPG Maker-specific data to engine-independent design.
- From static checklists to agent-assigned work orders with acceptance tests.

## Core Beliefs

1. The human creator remains the final creative authority.
2. AI agents work best when they have specialized roles.
3. Canon must be protected, but production must keep moving.
4. Game design should be represented independently of any engine.
5. Engine bridges translate design into implementation.
6. Every work order should produce a playable or verifiable improvement.
7. A game is not ready because files exist; it is ready because it plays well.

## AtlasStudio Owns

- Game vision
- Story canon
- World structure
- Character canon
- Agent roles
- Work order assignment
- Acceptance criteria
- Production readiness
- QA findings
- Cross-agent coordination

## AtlasStudio Does Not Own

- Raw RPG Maker map JSON
- Final tile placement for hand-authored maps
- Engine-specific IDs unless required by an engine bridge
- Final art assets
- Final audio assets
- Provider-specific hidden reasoning or chat history

## Production Standard

A completed AtlasStudio task should answer three questions:

1. What changed for the player?
2. How can another agent verify it?
3. What is the next playable improvement?
