# WO-0060 Visual Study: Temperate-Village vs. Coastal-Settlement Interior

Produced for `WO-0060` task 1 (contact sheets) and the visual half of task 3
(vocabulary definition) and task 4 (mapping onto Shop/Inn/House). This is a
visual-study document, not a palette ‚Äî it identifies and cites candidate tile
kinds/IDs by contact-sheet coordinate so that `WO-0060` task 2 (provenance-backed
palette tooling, owned by Codex) can build the actual verified `TilePalette`
instances. **Nothing in this document has been passability-, layer-, or
adjacency-checked against `Tilesets.json`** ‚Äî that confirmation is explicitly
task 2's job, not this one's. Per this project's own standing lesson
(`feedback_tileset_pixel_addressing`), every tile crop below was read directly
off a contact sheet rendered through the project's proven, `rmmz_core.js`-faithful
compositor (`render_map.py`/`contact_sheet.py`) ‚Äî never hand-derived pixel math.

## Source Material

The project's only general-purpose "building interior" tileset family is
`Inside_A1/A2/A4/A5/B/C` (`TheLastSwordProtocol-Game/img/tilesets/Inside_*.png`)
‚Äî the same family already used for Ashford's Elara House, Shop, and Inn.
`SF_Inside_*` (sci-fi machine interior, used for Sealed Node) and `Dungeon_*`
were not considered ‚Äî neither reads as a village dwelling. Contact sheets for
all six `Inside_*` sheets are in `contact_sheets/` alongside this file,
generated by `gen_inside_contact_sheets.py`'s scratchpad equivalent (see
Commands below).

## Finding: No Purpose-Built Coastal Props Exist

`Inside_B` and `Inside_C` (the two normal-tile/furniture sheets) were
inspected tile-by-tile. **No net, shell, anchor, driftwood, ship's-wheel, or
other explicitly nautical prop was found on either sheet.** This project's
asset library does not contain a dedicated "beach cottage" furniture set. This
is a real, disclosed limitation: the coastal identity defined below is built
entirely from **color and material substitution** (whitewash, pale stone,
blue-checkered tile, striped linen) using props already present on the
general-purpose `Inside` sheet, not from thematic prop replacement. A future
work order that licenses or commissions genuine coastal props would materially
strengthen this style pack; this one does not claim to be that.

### Not Used

`Inside_A1` is a cyan/turquoise pool-water autotile family (kinds 0-15) ‚Äî
visually striking but reads as a bath/fountain feature, not a floor or wall
material, and no archetype here has a water zone. Held out of both style
packs; a future "bathhouse" or exterior fountain archetype might use it.

## Mapping onto Shop / Inn / House (task 4)

No `GameplayGraph` was changed. Both style packs declare
`applicable_archetypes: ["ARCH-INTERIOR-SHOP", "ARCH-INTERIOR-INN", "ARCH-INTERIOR-HOUSE"]`
and speak the same `landmark_slots`/`modules_required` vocabulary those
archetypes already reference (`hearth_focal_feature`, `shelf_wall_run`, etc.)
‚Äî see each `building_archetype.json`'s `landmark_slots` and `StylePack.vocabulary.landmark`.

## Result

- `contract/examples/shared/style_pack_ashford_cozy_interior.json` ‚Äî extended

- Approval. Per `WO-0060`'s own Constraints and this project's standing convention, nothing here is `accepted` ‚Äî it is `generated_pending_review` at best, and task 6 requires Chris's live RPG Maker review before either pack is used in a production map.
